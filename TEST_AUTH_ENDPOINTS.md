# Testing Authentication Endpoints

## Issue Resolved

The problem was **token type mismatch**:
- ❌ You were using a **Firebase Admin SDK service account token** (from the JSON file)
- ✅ The endpoints need a **Firebase ID token** (user JWT)

## Solution

I've added a new testing endpoint that generates proper ID tokens for API testing.

## Setup

### 1. Add Firebase Web API Key to .env

Get your Firebase Web API Key:
1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project: `n8n-exchange-d9fef`
3. Go to **Project Settings** (gear icon) → **General** tab
4. Scroll down to **Web API Key**
5. Copy the key

Add it to your `.env` file:
```env
FIREBASE_API_KEY=your-web-api-key-here
```

### 2. Restart your server
```bash
cd backend
npm start
```

## Testing Flow

### Step 1: Get ID Token for Testing

**Endpoint:** `POST http://localhost:3000/api/auth/get-test-token`

**Body:**
```json
{
  "email": "your-registered-email@example.com"
}
```

**Example with curl:**
```bash
curl -X POST http://localhost:3000/api/auth/get-test-token \
  -H "Content-Type: application/json" \
  -d '{"email": "test@example.com"}'
```

**Response:**
```json
{
  "success": true,
  "message": "ID token generated for testing",
  "data": {
    "user": {
      "uid": "xyz123",
      "email": "test@example.com",
      "displayName": "Test User"
    },
    "idToken": "eyJhbGciOiJSUzI1NiIsImtpZCI6Ij...",
    "refreshToken": "...",
    "expiresIn": "3600",
    "note": "Use this idToken in Authorization header as 'Bearer <idToken>'"
  }
}
```

## Testing with Postman

### 1. Get ID Token
- Method: `POST`
- URL: `http://localhost:3000/api/auth/get-test-token`
- Body (raw JSON):
  ```json
  {
    "email": "test@example.com"
  }
  ```
- Copy the `idToken` from response

### 2. Test User Profile
- Method: `GET`
- URL: `http://localhost:3000/api/user/profile`
- Headers:
  - Key: `Authorization`
  - Value: `Bearer <paste-your-idToken-here>` (replace `<paste-your-idToken-here>` with actual token)

## Important Notes

1. **ID tokens expire** after 1 hour (3600 seconds). If you get "Token expired", request a new one.

2. **The test endpoint is for development only.** In production:
   - Frontend uses Firebase Client SDK
   - Client calls `signInWithEmailAndPassword()` or `signInWithCustomToken()`
   - Firebase automatically manages ID tokens
   - Client sends ID token in API requests

3. **Custom Token vs ID Token:**
   - **Custom Token:** Generated by backend, used by client to sign in to Firebase
   - **ID Token:** Generated by Firebase after sign-in, used for API authentication
   - `/login` endpoint returns custom token (for client-side sign-in)
   - `/get-test-token` endpoint exchanges it for ID token (for testing)

4. **Never use service account tokens** (from the JSON file) for user authentication. Those are for server-to-server communication only.

## All Available Auth Endpoints

1. `POST /api/auth/register` - Register new user
2. `POST /api/auth/login` - Login (returns custom token for client)
3. `POST /api/auth/get-test-token` - Get ID token for testing (NEW)
4. `POST /api/auth/google-signin` - Google authentication
5. `POST /api/auth/logout` - Logout user

## Protected Endpoints

All endpoints requiring authentication expect:
```
Authorization: Bearer <idToken>
```

Examples:
- `GET /api/user/profile` - Get user profile
- `PUT /api/user/profile` - Update user profile
- `GET /api/portfolio` - Get portfolio data
- `GET /api/portfolio/positions` - Get positions
- etc.
